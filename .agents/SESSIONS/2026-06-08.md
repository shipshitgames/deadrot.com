### Tier-sheet flow

```
codex exec "<structured $imagegen prompt>" -i <current-weapon-art>   (lower-right pose locked by reference)
   -> 1x5 PNG (base..evolved), chroma-key alpha
   -> cwebp lossless -> packages/assets/.../weapons/pyre/<w>-tiers.webp
   -> manifest weapon-<w>: { path, dimensions, scale(per-cell), tierSheet:{columns:5, tiers:[...]} }
runtime: activeWeaponVisualTier(score) -> weaponTierCellIndex -> tex.offset.x = cell/5 -> render one cell
         + TIER_GLOW[tier] tint + TIER_SCALE[tier] size on top
```

### Key decisions

- **codex is the only image generator available** (no fal/openai/replicate key in env or keychain). codex **cannot seamless-tile** (proven with a 2×2 tile demo) — so arena floor/wall textures CANNOT be AI-regenerated as-is; the trim+pad postprocess also breaks tiling.
- The "generate N cells together" trick (1×5 tier ladder) **works** even though seamless TILING doesn't — different problem (one escalating subject vs toroidal continuity).
- **Reference images + structured prompts** are the consistency fix: `codex exec "<PROMPT>" -i ref.png` (the GPT-Image-2 Task/Main subject/Composition/Style format). Text alone does not lock pose.
- Pose locked to the **existing lower-right angled** framing (user choice), anchored on shipped art so offsets stay roughly valid.

### Gotchas / mistakes

- `codex exec -i` is **variadic** (`<FILE>...`) — it eats a positional prompt placed after it. Put the prompt FIRST, `-i` LAST (or pipe via stdin). Decode webp refs to png (`dwebp`) first.
- Vite HMR got stuck showing a stale `WEAPON_TIER_SPRITE_IDS is not defined` error after multi-file edits; the console buffer never clears old entries — a hard reload fixes it; trust tsc/tests over the console history.
- assetgen's `generate --repo` writes `<repo>/src/assets/`, NOT the deadrot shared package layout (`games/<slug>/assets.json`). For in-place replacement: generate to `/tmp`, then move + hand-edit the manifest.
- Repo boundary: generation tooling lives in `../shipshitgames`; the asset pack stays in deadrot `packages/assets`. Memory updated: `deadrot-asset-generation.md`.
