### Gotchas / learnings (future sessions)

- **NEVER run `bun run check:fix` / `biome check --write .` broadly** — it runs
  the organizeImports assist REPO-WIDE and reformatted ~50 unrelated files once.
  CI's gate is `format:check` + `lint` only (NOT organizeImports). Use
  `biome format --write <specific files>` instead.
- **`UpgradeCard` / `MainMenuAction` ARE `<button>`s** — don't nest a button
  inside one. Make the card itself the button (its `onClick`).
- **`npm publish` is permanent** — publish from a merged canonical branch, not a
  feature branch (user published 0.2.0 from the feature branch; harmless here only
  because the bytes were identical).
- **Consuming a raw-TS package from npm**: vite/esbuild pre-bundles it fine — no
  vite config change needed. The Playwright suites are the real runtime proof
  (typecheck can't catch bundler / three-peer issues).
- **Cross-repo engine flow**: upstream to `../shipshitgames/packages/engine`
  FIRST → publish → bump the deadrot consumers. Both repos document it
  (`CANONICAL-ENGINE.md` + `repo-boundary.md`).
- The engine declares `three` as a **peer**, so each game keeps its single three
  instance (scourge 0.169 + vite `dedupe`, deadlane/warline 0.184) — no
  unification needed for the switch.
